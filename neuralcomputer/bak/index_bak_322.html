Neural Computer：第四个系统对象，正在出现

TL;DR: 它不是更强的 agent 或 world model，而是在押注 Neural Computer 会成为下一种机器形态。

arXiv Search GitHub

过去几十年，计算机逐渐成为人类完成任务的重要媒介。最近几年，AI 也开始进入这个位置：它不再只回答问题，还开始调用工具、操作界面、参与真实工作流。于是，一个更深的问题出现了：我们期待 AI 继续使用计算机呢？还是期待它最终成为一种计算机？

Neural Computer（神经计算机，NC）想讨论的，正是这个问题。它关心的，不是 AI 在现有计算机上把事情做得更多，而是模型能不能开始接手一部分原本属于机器运行本身的职责。

先澄清一下：这里说的 NC，不是 Alex Graves 那条 NTM / DNC 路线[1][2]，也不是在谈某种新硬件。本文更想讨论的，是一台 learning machine 会不会从“使用计算机”走向“成为计算机”。

它存在的价值不是更强的 agent，不是计算机环境的 world model，也不是传统计算机上外挂的一层智能。它真正关心的，是让原本常常外包给程序栈、工具链和控制层的那部分系统职责，开始进入模型实际依赖的 Runtime。我相信，这个设想其实早已在不少人的脑海里若隐若现；我想把这种状态称为“前共识”。但到现在，还没有一篇论文或博客把它讲得足够清楚、足够完整。

先说结论

  1. Neural Computer（NC）讨论的，不是“怎样让 agent 更强”，也不是“怎样做一个模拟计算机的世界模型”。 它真正追问的是：模型能不能承担计算机 Runtime 的核心职责。
  2. 它和另外三个系统对象并列。 传统计算机负责稳定执行，Agent 负责在既有计算机上完成任务，World Model 负责学习环境动态，而 NC 试图让 learning machine 本身成为运行中的机器。
  3. Completely Neural Computer（完备神经计算机，CNC）是 NC 的完成态。 能力可以被安装、稳定复用、显式更新，并且变化可以被追踪、回放和回滚。
  4. 当前原型已经开始证明一部分早期 Runtime 原语。 尤其是 I/O 对齐和短时程控制；但复用、治理、一致性和符号稳定性还远没有真正建立起来。
  5. 真正的门槛不是下一次系统展示。 而是 run / update separation 能否成立，以及状态边界、更新边界和证据边界能否被明确写进 Runtime。
  6. 如果这些条件成立，讨论的就不再只是“更强的 agent”。 learning machine 本身会开始成为“计算机”。

1）为什么是现在：“一种新的系统对象”正在出现

今天同时发生着三件事。

第一，agent 变得越来越会做事。 从 2023年的 MetaGPT (“古早” Coding Agents之一)[3] 只能勉强写出几百行代码，到2025年 Cursor、Codex 和 Claude Code 已经成为程序员生产力标配工具，再到今天 OpenClaw[4] 真正走入大众视野，大家关心的已经不只是 agent 偶尔能把一件事做成，而是它能不能进入真实生产或日常生活，并稳定、持续地替你处理一系列或琐碎或高级的人类事务。

对 agent 来说，共识是：bottleneck 在于（1）长时程任务怎样保持稳定，（2）能力怎样沉淀，（3）流程怎样持续复用。眼下的解决路径，仍主要是在 agent 的 scaffold（或 harness）一侧做加法：用更强的记忆、更长的工作流和更稳的行动闭环，尽可能把事情做成。再往前推，更激进的一条路径是递归式自我改进：模型开始训练下一代模型，agent 开始持续改写自身[5]。

第二，world model 变得越来越擅长建模动态环境。 它本来就指向环境演化；更值得注意的是，这种能力开始进入真实系统。尤其在现实里难以低成本、反复采集的 corner case 上，rollout 正被直接用进预测、规划、控制和训练。沿着这条线看，从 Jürgen Schmidhuber 在 1990 年提出的 Making the World Differentiable[6]，到 David Ha 和 Jürgen Schmidhuber 在 2018 年的《World Models》[7]，再到现在 Waymo 把 world model 用进自动驾驶仿真与训练[8][9]，这条路线已经不只是研究方向，而是在变成系统能力。

对 world model 来说，关键不是把任务做成，而是把未来先展开。它补的不是执行本身，而是系统的可推演性：采取某个动作，环境会往哪里走；即使不直接行动，哪些未来也值得先生成、先试验、先暴露风险。今天这条路线已经分出几种明显方向：在自动驾驶和 physical AI 里，它更像一种仿真与合成数据引擎，用来补足真实世界里昂贵、危险或稀缺的数据，例如 Waymo World Model 和 NVIDIA Cosmos[8][10]；在 spatial intelligence 里，它开始追求可生成、可进入、可持续交互的 3D 世界，例如 World Labs 的 Marble[11]；在更偏实时互动世界的方向上，生成模型已经开始从静态内容生成走向可控、可交互、可探索的环境生成，代表性例子包括 GameNGen 对 DOOM 的实时神经模拟[12]，以及 Google DeepMind 的 Genie 2 / Genie 3[13][14]。这些方向看起来差异很大，但本质上都在做同一件事：把对环境未来演化的模拟，变成系统内部的一种能力。

第三，传统计算机在 AI 时代的结构性摩擦越来越明显。 今天大量任务都是开放式的、长时程的、持续交互的。传统软件栈依然极其强大，但在许多以自然语言、示范、界面操作和弱约束为主的场景中，它已经显得笨重。

传统计算机本身也在为 AI 重写底座。芯片、编译器、内存系统、软件栈都在变得更 model-friendly。但这些变化多数仍然发生在既有计算范式内部：它们让旧机器更适合 AI，却没有改写“机器是什么”。像 Taalas 这样的路线则更进一步，开始把特定模型做成一种部署单元：模型不再只是跑在机器上的负载，而是在逼近“按模型组织硬件”这条线[15]。但至少今天，这还只是推理部署单元层面的变化，还谈不上一个已经成立的通用新系统对象。

把这三件事放在一起，问题就更具体了：如果 agent 在变得更会做事，world model 在变得更会推演，而传统计算机也在为 AI 重写底座，那么会不会出现一种新的 Runtime，把执行、rollout 和能力沉淀统一进同一台 learning machine 里？

从关系结构上看，这对应的是人和机器之间主关系的一次迁移：在传统计算里，人主要和 computer 交互；到了 agent 时代，人更多是和 agent 交互，再由 agent 去调用 computer 把事情做成。world model 更像一个并行的预测层：它既可以服务于 human，也可以服务于 agent，但本身不负责把事情做成。再往前推，NC 要改的就不再是中介，而是机器本身：它不是在人和 computer 之间再加一个更强的 agent，而是试图把今天分散在 computer、agent 和 world model 之间的职责，收拢到同一台 learning machine 内部。那时，人面对的就不再只是“agent 代替自己调用 computer”，而是直接使用这样一台神经计算机。

这也说明，交互本身会开始带有“编程”的意味。今天，自然语言指令、键鼠轨迹、屏幕变化和任务反馈，大多只是过程日志；在 NC 的设定里，它们会变成塑造未来行为的材料。今天我们主要通过代码安装能力；以后，示范、交互轨迹和约束本身，也可能成为能力进入 Runtime 的入口。

2）什么是 NC，什么又算 CNC？

先看一张表：它把传统计算机、Agent、World Model 和 Neural Computer 放到同一把“尺子”上比较。看完这张表，大家应该就能一眼看出它们的区别和联系：它们各自围绕什么组织，source of truth 主要落在哪里，又分别承担什么职责。

系统对象 围绕什么组织 source of truth 主要在哪里 主要职责
传统计算机 程序 程序与程序栈 稳定执行程序
Agent 任务 外部环境、工具链与工作流 在既有环境中完成任务
World Model 环境演化 状态演化模型 预测与推演环境变化
Neural Computer Runtime Runtime 里的能力与状态 让机器持续运行、沉淀能力并治理更新

这张表已经很直观了，我就不再用一大段抽象描述去重复它。接下来不如直接设想一下：如果 NC 已经存在，人会怎么使用它？对传统计算机，你安装的是软件；对 agent，你描述的是任务；对 NC，你更像是在给机器安装能力，并期待这些能力以后继续留在机器里。

如果它成立，机器会长得像什么？

第一，它未必会继续朝“更像大脑、更像感知模型”的方向长。 主流模型作者通常喜欢把自己的结构说得更像人类大脑，或者模拟动物感知。但在我的想法里，当规模继续上去，NC 更可能朝更稀疏、更可解释、更像 circuit 的方向演化。未来的 CNC 也许不是一团难以追踪的连续表征，而会更像一个可路由、可组合、强解释性的逻辑底座。它不一定模仿动物感知或人脑，反而可能更接近一种带有 NAND 气质的神经网络：离散、稀疏、局部可验证。至少目前，这条路还没有被系统展开；但我有点惊喜地发现，OpenAI 最近在 weight-sparse transformers 上做的一些工作，至少说明了一件事：把神经网络做得更稀疏、更局部、更可路由，未必只是为了好解释，它本身就可能是一条机器式架构路线[16]。

第二，它也未必总靠整体改参数来升级自己。 更自然的图景是：用户输入不只触发一次性行为，而是逐渐安装、调用、组合并保留可复用的 neural routines，甚至形成以后还能继续调用的内部 executor。至少在功能分工上，这更像传统计算机里的“内存”而不是处理器：升级未必意味着重写整台机器的本体，也可能只是把这些新结构稳定写进一层可寻址、可调用、可保留的内部状态。沿着这条线往前推，升级不再只是“换一个更大的 model”，而更像是在机器内部持续安装新部件。有意思的是，若干年前的 NPI 和 HyperNetworks 可以被看成一些相似但还不完整的早期思路：前者试图把复杂程序拆成可调用、可组合的子程序[17]；后者则提示，机器甚至可能继续生成下游 neural modules，去扩展自己的能力边界[18]。当然，我认为野心可以更大一点，一个足够强的 Neural Computer，完全可能直接生成新的 (sub-)NNs，再把它们以可插拔的方式挂进自身内部，像今天安装或卸载软件一样自然，只是这一次省掉了手写代码和编译这一层中介。

第三，它还可能把 world model 式的 rollout 逐渐收进 Runtime 里。 到那时，rollout 不再只是外部的训练工具或规划模块，而会开始变成机器的日常机制。人类可以给出输入、期待的输出（GT），也可以只提前写好评估指标；甚至在某一轮里什么都不再给，Runtime 也可以在内部持续 self-play、自测、筛选和压缩候选做法，再把有效改进沉淀成下一轮能力更新。理想状态下，人去睡觉的时候，机器并不是停着的；它还在内部完成评估、试错和自我迭代。当然，这一切的前提不是放任系统偷偷变化，而是 update 路径本身可被治理。

这么一放，NC 的对象感其实就出来了：关键不在于它一次能做多少事，而在于能力是否真的开始住进 Runtime，并在那里被安装、复用、执行和治理。CNC 则是这个想法的完成态。按原论文的定义，一个 NC 实例只有在同时满足四个条件时，才可以算作 CNC：它必须是 Turing complete、universally programmable、除非被显式重编程否则保持 behavior-consistent，并真正体现 NC 相对传统计算机的架构与编程语义。下面这张表，就是对原论文这四条要求的一个更直白的总结。

CNC 条件 更直白地说 工程上大概要看到什么
Turing complete 不是只能完成几类固定任务，而是在原则上具备通用计算的表达能力。 但“可表达”不等于“可执行”：真正要看的是，随着有效 memory 和 context 增长，同一个 NC 能否稳定承接更长、更复杂的算法过程，而不是任务一拉长就换一种失效方式。
Universally programmable 输入给它的，不该只触发一次行为，而应能真正安装成以后还能调用的 routine 或 executor。 能力可以被安装、调用、组合、保留；一旦进入 Runtime，就能跨任务稳定复用，而不是每次重新教一遍或重新外包给工具链。
Behavior-consistent 日常使用不应偷偷改变机器；行为变化只能来自显式更新。 同版本行为可复现，执行与更新轨迹都可追踪，出了问题能 replay / rollback，长时程 drift 可以被测量和治理。
Machine-native semantics 它不只是用神经网络去模仿旧计算机，而是开始长出自己的机器语义和编程方式。 神经底座能靠组合、路由、连续状态和内部执行结构带来传统栈不擅长的能力；同时，instructions、demonstrations、traces 和 constraints 本身开始成为编程入口，而不再只有手写代码。

3）为什么论文当前原型会长成这样，以及它到底证明了什么

我真正期待的 Neural Computer 周期，大概要再过三年才会到来。而在那之前，更需要有人先把 Neural Computer 讲清楚：它到底是什么，我们又为什么需要它。也正因为如此，前面的 1）和 2）章节，其实都走在当前 3）这一原型前面。我先叠个甲：论文里用视频生成模型来建模 NC，大概率不是 NC 的终局形态。但放在今天，它仍然是我认为最快把像素、动作和时间 rollout 统一进一个端到端原型的路径。换句话说，它更像眼下最顺手的统一载体。

一个有趣的事

这个小故事和 Neural Computer 本身无关，但能从侧面说明一件事：从“前共识”走到被广泛接受，往往需要一段不短的沉默期。我自己一直受 Gödel Machine 和 ultraintelligent machine 这些想法影响[30][29]。所以在 2023 年，尽管那时基础模型的 intelligence 还远远不够，Coding Agent 也还只是萌芽期，我就有一个很强的判断：Recursive Self-Improvement 和 Learning from Experience 会托基础模型和 Coding Agents 的福，真的实现。我还是把这段判断写进了 MetaGPT outlook。

后来，我自己也贡献了 GPTSwarm 和 Agent-as-a-Judge 这些相关工作[24][25]；也将在 ICLR 2026 举办第一届 AI with Recursive Self-Improvement 研讨会[27]。今天回头看，这条判断之所以会在 2025 到 2026 密集浮到台前，一方面是因为 Learning from Experience 被 David Silver 和 Richard Sutton 在 2025 年更明确地命名并推到前台[19]；另一方面，则是因为 Recursive Self-Improvement 在这段时间里被顶咖讨论得越来越具体：

Sam Altman 在 2025 年 6 月把“用现有工具帮助我们找到新的科学洞见，并帮助我们创造更好的 AI 系统”称作 recursive self-improvement 的 larval version[20]；Dario Amodei 在 2026 年 1 月把它写成一个已经启动的自我加速闭环，并判断 1 到 2 年内就可能走到“当前一代 AI 自主构建下一代”[21]；Demis Hassabis 在同月的 Davos 对谈里更谨慎一些，把问题收束到自我改进闭环能否在没有人类介入的情况下真正闭合，以及这种系统本身的风险[22]；Elon Musk 的公开表述则更激进一些，更接近把它看成一个已经开始滚动的自增强过程[28]。

但在 2023 年，它们（RSI, Learning from Experience）还远没有成为主流叙事。也正因为有这个时间差，我会觉得今天的 Neural Computer 也处在一个相似的周期里。

我们正式进入这一章节的正题。先别急着分析方法，请读者先把下面这组视频当成偶然刷到的终端录屏去看。如果在第一眼，尤其是不认真看的那几秒里，你已经有一点“这像是真的”感觉，那么这篇论文想建立的第一层直觉，其实就已经成立了。

因为作者并不是一上来就要证明“神经计算机已经会推理、会编程、会长期运行”这么重的命题；他们先做的，是把最外层的 runtime 表象稳住，让模型先学会屏幕上那些最容易被人忽视、却最构成“这就是一台机器正在运行”的东西：终端几何、ANSI 配色、光标显隐、输入节奏、长日志滚动，以及动作之后界面怎样继续往前走。

训练方式也故意保持得很朴素：底座就是 Wan2.1[31]，CLI 这边把任务当成 text-and-image-to-video，用 caption 和首帧去驱动 latent rollout；GUI 这边则在同一个 backbone 上额外接入 action encoder，并把动作以不同深度注入 transformer。也正因为框架没有玩太多花活，后面几组结果里真正起作用的东西才更容易被看清。更重要的是，作者不是只放一组“很像”的 demo 就收手，而是故意搭了三层台阶：先用公开终端回放把开放世界终端动态学出来，再用显式脚本终端轨迹把动作因果钉牢，最后推到真实 GUI 桌面，去看点击、悬停、键盘输入和窗口切换能不能真的驱动状态变化。下面就沿着这三层往上走。

CLIGen (General)

1）先看 CLIGen (General)：不认真看时，已经有点以假乱真

这一组最适合先用肉眼扫过去。如果我不先说这是生成视频，在不认真看的时候，它已经很像真实的终端录屏；真正该盯的，是终端几何、配色、光标和滚动节奏能不能一直稳住。

最外层这组对应的是 CLIGen (General)：大约 82 万段公开 asciinema .cast 轨迹，被用官方工具回放，再用 agg 和 ffmpeg 渲染成视频。这里保留的不是抽象“终端语义”，而是终端真正让人信服的那些细枝末节：palette transition、cursor state、terminal geometry、长日志滚动、全屏 TUI、进度条和状态栏。作者还把底层 buffer、时间轴和键盘事件对齐起来，再自动补齐多档文本描述，去给这些片段生成 prompt。

为什么一定要先看这一步？因为它最像“先把读者骗过第一眼”。如果 CLIGen (General) 都做不到，后面再谈动作、控制、乃至 Neural Computer 都会显得太早。但它一旦成立，你就会意识到：模型至少已经开始把开放世界终端的外部动态压进自己的 rollout 里了。只是到这一步，我们还不能说它真的“理解”了终端，只能说它先学会了终端作为 runtime 的表象。

CLIGen (Clean) / Math

2）再看 REPL 和 Math：它不只是在“画终端”

再往下一层看，就不是“像不像终端”了，而是它有没有一点真正的 REPL 感：输入、编辑、回车、回显，以及一些简单 math 过程能不能被承接。

但如果只停在 CLIGen (General)，你完全可以说：这还只是把终端“画得更像”。所以作者又单独做了 CLIGen (Clean)：用开源 vhs 工具把终端交互写成脚本，在 Docker 环境里重复执行，得到显式动作、节奏更干净、因果关系更硬的终端轨迹。这里大约写了 25 万条脚本，过滤后保留下来两块数据：约 7.8 万条 regular traces，以及约 5 万条 Python math validation traces；caption 也不再靠模型脑补，而是直接从脚本本身导出。

这一步的意义，是把问题从“像不像一段终端录屏”推进到“它有没有一点真正的 REPL 感”。你现在看的不该只是配色和滚动，而要去盯输入、编辑、回车、回显、报错、以及局部字符更新是不是能被动作带着走。换句话说，CLIGen (Clean) 不是为了让 demo 更花，而是为了把 action-to-pixel causality 钉住。

结果也很有意思。OCR character accuracy 从 0.03 拉到 0.54，exact-line 到 0.31，说明字符已经开始往对的位置落；但如果把问题逼到最硬的符号层，CLI 算术原生只有 4%，靠 reprompt 才能到 83%。这恰好给出一个很诚实的结论：当前原型已经不只是“会画终端”，但也远远不能说它已经长出了可靠的原生符号能力。它更像是在把交互形态和局部因果先学进去，而不是已经把推理底座一并解决了。

GUIWorld

3）再看 GUIWorld：界面操控也开始成立

到了 GUI，真正要看的也不是画面顺不顺眼，而是点击、悬停、切换窗口、编辑字段之后，界面有没有真的往对的状态走。

真正让这条线从“终端视频生成”往“运行中的机器”再推一步的，是 GUIWorld。因为到了 GUI，终端里很多可被文本吸收的结构都消失了，你面对的是更脏、更连续、也更依赖精确时序的世界：鼠标移动要落点，悬停要出反馈，点击之后按钮、下拉框、模态窗和输入框都要真的变状态。

GUIWorld 的数据也因此更像一个完整的交互 rig：作者在 Ubuntu 22.04 的 XFCE4 桌面里固定了 1024×768、15 FPS 的环境，用 Xvfb 和 VNC/noVNC 跑桌面，录屏用 mss 和 ffmpeg，动作回放与日志用 xdotool。数据分成三块：约 1000 小时的 Random Slow、约 400 小时的 Random Fast，以及约 110 小时 goal-directed supervised trajectories。前两者负责把鼠标加速度、停顿、悬停、窗口切换这些开放世界噪声喂给模型，后者则给出更清晰的 action-response 对，让模型真正学会“做了这个动作之后，界面接下来该怎么变”。

更关键的是，作者基本把今天几类主流动作注入思路都试了一遍。external 是最浅的一种，只在输入侧调制 latent；contextual 把动作 token 并进主序列里，和 World and Human Action Models 这一类 token-based world model 很接近[33]；residual 则像 ControlNet 一样，在 backbone 旁边挂一条 residual branch 来推动作信息[34]；而 internal 是把 action cross-attention 直接插进 transformer block 内部，这一点和 Matrix-Game 2.0 的 multi-stream injection 思路最像[32]。这几种设计的差别，不只是“有没有看见动作”，而是动作到底在多深的层里、以多核心的方式参与状态演化。

最后的心得也很清楚。四种模式里，internal 的综合结果最好：SSIM 0.863、FVD 14.5，说明对于 GUI 这种细粒度、强时序、强局部交互的环境，把动作直接打进 block 内部，比只在输入层改一把，或者像 ControlNet 那样在外面补一条残差分支，更容易让模型真正把“动作之后界面怎样继续走”学进主干里。作者还把 raw-action 和 meta-action 两种编码单独比过，后者略好，但差距远小于注入位置本身。与此同时，110 小时的 supervised data 明显好于约 1400 小时 random data，显式光标视觉监督又把 cursor accuracy 直接拉到 98.7%。这说明 GUIWorld 给出的最重要结论不是“桌面终于生成得很漂亮”，而是另一件更关键的事：GUI 这条线真正缺的不是更多随机视频，而是更硬的动作语义、更明确的状态转移、以及把光标当成视觉对象来监督。

4）最后再回到论文：它已经证明了什么，还差什么

如果按 CLIGen (General) 到 CLIGen (Clean) 再到 GUIWorld 这一条线看，这篇论文真正漂亮的地方，不是某一个 demo 特别像，而是它把问题拆成了三层越来越难的命题，并且每一层都各自给出了一点成立的证据。第一层，开放世界终端动态可以被压进 rollout 里；第二层，显式动作开始能落到字符编辑、回车回显和局部 math REPL 这种更硬的交互上；第三层，GUI 里的点击、悬停、输入和窗口切换，已经开始出现可辨认、可量化的 post-action state change。

但它真正证明的，也就到这里为止。它证明了像素、动作和时间 rollout 可以先被统一进同一个端到端原型；证明了动作注入的位置、数据质量、以及显式 cursor supervision 会强烈影响一个“神经 runtime”能不能站稳；也证明了如果你想让模型更像一台运行中的机器，光靠把视频做得像还不够，必须把 action semantics 一起压进去。可它还没有证明，我们已经拥有了一台可长期运行、可稳定复用、可治理更新、原生符号可靠的 Neural Computer。

所以我看完这篇论文之后最大的心得反而不是“CNC 已经来了”，而是“接下来最该补的短板已经暴露得很清楚了”。CLI 上最薄弱的是原生符号能力，GUI 上最薄弱的是长期一致的动作-状态对齐，而两者共同缺的，则是 installation、reuse、update governance 这些真正让 runtime 变成 machine 的东西。算术原生只有 4% 这件事，其实不是一个小瑕疵，而是在提醒我们：系统杠杆已经出现了，但 machine-native intelligence 还远没有站稳。

也正因为如此，这篇论文最有价值的地方，不是提前宣判 Neural Computer 已经成立，而是把下一步的施工图画出来了：先用更高质量、目的明确的交互数据把动作语义钉牢；再把像 internal 这样更靠近 backbone 核心的注入方式做深做稳；然后才谈更强的记忆、更新、复用和治理。它真正照亮的，不是终局，而是通往终局时哪几步最该先做。

4）如果 NC 成立，软件、硬件和“程序”都会改变

最后，如果 NC 真的成立，变化就不只发生在某个工具栈里，还会一路传到程序对象、软硬件接口和机器边界本身。至少有三件事会跟着变。

1）被安装的对象会改变

今天我们安装的是代码，写的是指令序列；到了 NC 的设定里，被安装、复用和版本化的对象，会逐渐从代码转向能力本身。代码不会立刻消失，但它会失去“唯一入口”这层特权。instructions、demonstrations、traces 和 constraints 都可能直接变成安装材料；而有些能力，未来甚至可能以内置 executor、内部 routine 或被编译进神经底座的结构存在，而不再总要先翻译成外部工具链里的代码。软件生产也会从“先写程序，再执行程序”转向“先塑形能力，再让机器长期调用它”。到那时，“程序”不再只是一段待执行代码，而会越来越像一种可被安装、组合、更新和治理的能力对象。

2）架构范式会改变

如果 Runtime 真开始承接能力安装、复用和治理，那么今天的软件栈和硬件栈也会跟着变。第 2 节已经提过，NC 可能会长成更稀疏、更可路由、更 machine-native 的神经底座，也可能通过可插拔的 neural routines 持续扩展自己；同一台机器内部，甚至可能并存擅长感知与 rollout 的路径，以及更适合精确执行与结构化计算的路径。落到工程上，变化就不只在模型本身，还会波及围绕它的 memory、routing、scheduling、verification、logging、replay / rollback，乃至整套软硬件协同方式。硬件不再只是给某个模型推理加速，而会开始为一台持续运行、持续安装能力、持续治理更新的机器服务。换句话说，系统设计的重心会从“怎样把一个 model 跑快”转向“怎样让一台 learning machine 长期稳定地运行”。

3）计算机的物理边界会变松

“电脑在哪里”这个问题会变得次要。高性能的神经计算机会越来越像一种持续在线的能力基础设施：用户不必先承担高昂硬件成本，也能直接使用足够强的神经计算能力；被服务化的，不只是存储和算力，还有 Runtime 本身、能力对象和行为规则。更重要的是，多个入口会开始接入同一台持续运行的机器：手机、浏览器、终端、IDE，甚至未来的具身设备，都可能只是在接入同一个 NC，而不再各自对应一套割裂的软件实例。

所以真正押注的，我想再重申，不是 agent 再强一点，也不是 world model 再往前推一点，而是 Neural Computer 会重新定义我们所说的“计算机”。到那时，被安装、调用和治理的对象都会往机器本身收拢；被改写的，也不只是某个工具栈，而是“计算机”这个词本身。
