Neural Computer

Blog Authors Mingchen Zhuge

This is not another agent demo. Neural Computer is a bet that one learned runtime can absorb compute, memory, and I/O and behave like a living computer.

In the paper, we define a Neural Computer (NC) as a learned, executable, and eventually programmable system where model, environment, and runtime share one latent state. The long game is even bigger: a Completely Neural Computer (CNC), where the classical stack is no longer bolted together module by module.

We instantiate this in two directions: NC-CLIGen for terminal worlds and NC-GUIWorld for desktop interfaces. Same core idea, different surfaces: read recent pixels, consume action streams, roll out the next interface frame.

Why This Is A Big Deal

Classical systems split responsibilities across layers: one subsystem computes, another stores state, another mediates interaction. Neural Computer collapses those boundaries into an update-and-render loop. It is a conceptual shift from “model as tool” to “model as runtime.”

On CLI, this already looks real: structured rollouts, coherent terminal physics, multi-step command behavior, and strong character-level rendering when data is clean and conditioning is explicit.

100 tasks sampled from a 1,000-problem holdout. Accuracy jumps from 4% to 83% via stronger conditioning.

That jump is exactly the point: system-level conditioning can unlock major symbolic gains without changing the backbone. But it also tells us to be honest about what is happening: better interface specification and answer-conditioned rendering are not the same as proven native computation.

Neural Computer is where a video model starts to look less like a renderer and more like an executable interface runtime.

GUI Is Where It Gets Serious

In GUIWorld, we evaluate multiple action injection strategies and both raw-action and meta-action encodings. The most striking finding: explicit visual cursor supervision changes the game.

Position-only supervision reaches 8.7% cursor accuracy, enhanced position features reach 13.5%, and explicit cursor-visual supervision jumps to 98.7%. Another strong signal: about 1,400 hours of noisy exploration data still lose to around 110 hours of high-quality goal-directed trajectories (Claude CUA subset). Data quality dominates raw volume.

Toward CNC

We are still far from fully neural computers. Long-horizon layout stability, fine-grained text/icon fidelity, and verifiable in-runtime computation are open challenges. But the trajectory is now concrete, not speculative.

The roadmap is explicit: push compute, memory, and I/O into one learned runtime, then scale from interface imitation to reliable execution. If this works, we will not just have better assistants. We will have a new software substrate.

What Comes Next

  • Scale high-quality interaction data instead of brute-force random traces.
  • Move from answer-conditioned rendering to verifiable native computation.
  • Unify CLI and GUI prototypes on the path to a single CNC runtime.
